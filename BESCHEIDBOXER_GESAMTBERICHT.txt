TEIL 1: EXECUTIVE SUMMARY
================================================================================

BescheidBoxer ist eine ambitionierte React/TypeScript-Webanwendung, die als
KI-Assistent Menschen bei der Pruefung und Anfechtung fehlerhafter Bescheide
im deutschen Sozialrecht unterstuetzt (Buergergeld, ALG I, Sozialhilfe).

TECH-STACK: React 18 + TypeScript + Vite + Tailwind CSS + Supabase + Vercel
KI: Anthropic Claude API (claude-sonnet-4-5)
MONETARISIERUNG: Freemium + 4-Tier-Abo-Modell (0 / 2,99 / 4,99 / 7,99 EUR/Monat)

STATUS: Die App ist live und funktional, befindet sich aber in einem
"Extended Beta"-Zustand. Viele Features nutzen Demo-Daten statt echter
Backend-Integration. Die KI-Funktionen (BescheidScan, Chat, Brief-Generator)
sind teilweise nur Mockups.

GESAMTBEWERTUNG: 5.8/10 (gutes Konzept, solider Code, aber noch nicht
production-ready fuer ein Finanzprodukt)

2.2 SICHERHEITSAUDIT
---------------------

KRITISCH - Sofort beheben:

1) CREDITS NUR CLIENT-SEITIG VALIDIERT
   Datei: src/contexts/CreditsContext.tsx
   Problem: Credits werden nur im Browser-State (useState) verwaltet.
   Kein Server-seitiger Check bei API-Aufrufen.
   Risiko: User kann Credits per DevTools manipulieren und unbegrenzt
   die KI-API nutzen --> unkontrollierte Kosten!

2) SUPABASE-KEYS ALS FALLBACK HARDCODIERT
   Datei: src/integrations/supabase/client.ts
   Problem: URL und Anon-Key sind als Fallback im Code.
   Risiko: Bei fehlender .env verbindet sich die App mit dem falschen Projekt.

3) KEINE RATE-LIMITING FUER ANTHROPIC API
   Datei: api/amt-chat.ts
   Problem: Jeder Request wird direkt an die Anthropic API weitergeleitet.
   Kein Limit pro User, pro IP, pro Zeitraum.
   Risiko: Ein einzelner User kann tausende Anfragen senden --> massive Kosten.

4) PASSWOERTER IM LOCALSTORAGE (DEMO-MODE)
   Datei: src/contexts/AuthContext.tsx (Zeile 182-194)
   Problem: Im Demo-Mode werden Passwoerter im Klartext im localStorage gespeichert.
   Risiko: Jeder mit DevTools-Zugriff kann Passwoerter auslesen.

HOCH - Zeitnah beheben:

5) WEBHOOK OHNE DEDUPLICATION
   Datei: api/amt-webhook.ts
   Problem: Wenn Stripe einen Webhook 2x sendet (kommt vor), werden
   doppelte Credits hinzugefuegt. Keine Idempotency-Pruefung.

6) FORUM-RLS UNVOLLSTAENDIG
   Problem: Authenticated Users koennen Posts erstellen, aber es wird
   nicht geprueft, ob sie ihre User-ID manipulieren.

7) KEINE INPUT-SANITIZATION
   Problem: Viele Formulare akzeptieren Freitext ohne Bereinigung.
   Potenzielle XSS-Risiken (z.B. NotizenPage.tsx).

7.1 RECHTSDIENSTLEISTUNGSGESETZ (RDG)
---------------------------------------

ACHTUNG: BescheidBoxer bewegt sich in einer rechtlichen Grauzone!

- Paragraph 2 RDG: "Rechtsdienstleistung" = jede Taetigkeit in konkreten
  fremden Angelegenheiten, die eine rechtliche Pruefung erfordert
- Paragraph 6 RDG: Unentgeltliche Rechtsdienstleistungen sind erlaubt,
  wenn sie unter Anleitung einer qualifizierten Person erfolgen

EMPFEHLUNG:
1. Prominenten Disclaimer einbauen: "BescheidBoxer bietet
   Rechtsinformationen, keine Rechtsberatung."
2. Bei konkreten Einzelfall-Empfehlungen IMMER auf Anwalt/Beratungsstelle
   verweisen (ist im Chat-Prompt bereits vorgesehen - gut!)
3. Rechtliche Pruefung durch Anwalt fuer Sozialrecht einholen
4. Ggf. Kooperation mit einem Anwalt als "Aufsichtsperson" nach Paragraph 6 RDG
